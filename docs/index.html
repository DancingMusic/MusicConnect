MusicConnect 连接器 API 开发规范

本页是 DancingMusic 音乐连接器的统一开发入口。它只描述接口、仓库边界、演示页和发布流程； 具体实现必须放在独立的 MusicConnect-* 仓库中，并由宿主通过固定 tag 的 ESM bundle 动态加载。

总览

DancingMusic/MusicConnect 是统一文档站，不再作为某个音乐平台的实现仓库。 网易云、iTunes、Archive、Radio、QQMusic、Spotify 等实现各自维护在 DancingMusic/MusicConnect-<Name> 仓库中。

i
核心边界 连接器负责搜索、歌单、曲目详情、真实音频源、歌词和配置 schema；宿主负责播放器、队列、UI、缓存和动态加载。

仓库边界

仓库职责禁止
DancingMusic 宿主 UI、播放器、连接器 registry、CDN mirror fallback、默认连接器清单 直接 import 某个 MusicConnect-* 实现
MusicStore MusicConnector 类型、曲目/歌单/歌词数据结构、registry primitives 包含任何平台实现或宿主 UI
MusicConnect-* 单一平台实现、契约测试、dist/index.js、仓库 README、Demo Page 控制播放器、渲染宿主 UI、访问 Dance 插件或硬编码用户凭证

快速开始

开发连接器时，先在你的独立 MusicConnect-* 仓库安装 MusicStore SDK 合约。 MusicConnect 本仓库只作为文档入口，不作为多个平台实现的安装包。

npm install 使用方式

当前 SDK 尚未发布到 npm registry，可以先通过 GitHub 源安装。等 @dancingmusic/music-store 发布后，再切换到 registry 包。

bash — npm install 
npm install github:DancingMusic/MusicStore#main
npm install -D typescript tsup vitest
bash — registry package once published 
npm install @dancingmusic/music-store
typescript — src/index.ts 
import type { MusicConnector } from '@dancingmusic/music-store';

export default class YourConnector implements MusicConnector {
  readonly meta = {
    id: 'your-source',
    name: 'Your Source',
    version: '0.1.0',
    capabilities: ['search', 'stream'] as const,
  };

  async init(config: Record<string, unknown>) {}
  async search(query: string) { /* return playable tracks */ }
  async getStreamUrl(trackId: string) { /* return stream info or null */ }
}
  1. 创建独立仓库 使用 DancingMusic/MusicConnect-<Name> 命名。不要把多个实现合并进一个仓库，也不要把实现作为主仓 submodule。
  2. 实现 MusicStore 接口 peer-depend @dancingmusic/music-store，默认导出连接器 class，并按 capability 实现 search、stream、playlist、lyrics。
  3. 提交构建产物和测试 使用 tsup --format esm --target es2020 --no-splitting 输出 dist/index.js，并保留 vitest 契约测试。
  4. 发布 Pages Demo 在 docs/index.html 中提供搜索、播放、歌词状态和配置提示，从 main:/docs 发布到 GitHub Pages。

MusicConnector 接口

连接器需要暴露稳定 metadata，并确保返回的数据可以被宿主直接消费。

typescript 
import type { MusicConnector } from '@dancingmusic/music-store';

export default class YourConnector implements MusicConnector {
  readonly meta = {
    id: 'your-source',
    name: 'Your Source',
    version: '0.1.0',
    capabilities: ['search', 'stream', 'playlist'] as const,
    configSchema: [
      { key: 'apiBaseUrl', label: 'API Base URL', type: 'url', required: false },
    ],
  };

  async init(config: Record<string, unknown>) {
    // token / endpoint / region only comes from runtime config
  }

  async search(query) { /* return playable tracks */ }
  async getTrack(trackId) { /* return track detail */ }
  async getStreamUrl(trackId) { /* return url or null */ }
}

Track ID 必须带平台前缀，例如 netease:123456、itunes:1440857781。 getStreamUrl() 不得返回未验证的外链；不可播放时返回 null。

配置声明

需要 endpoint、token、OAuth token URL 或其它运行时参数的连接器，必须通过 meta.configSchema 声明字段。宿主会根据 schema 渲染添加/编辑表单。

字段类型要求说明
key 必填string稳定写入 connector config 的字段名
label 必填string可读宿主表单显示名
type 必填text | url | password | select准确影响输入控件和校验
required 可选boolean默认 false必填参数必须声明
placeholder 可选string推荐给出正确格式，不写真实凭证

能力矩阵

capabilities 必须只声明真实实现的能力。宿主依据它决定是否展示搜索、歌单、歌词等入口。

能力对应方法契约测试
searchsearch(), getTrack()mock API 下返回带平台前缀的曲目
streamgetStreamUrl()成功 URL 与 null-stream 两条路径都覆盖
playlistlistPlaylists(), getPlaylistTracks()分页、排序、空歌单和曲目形状
lyricsgetLyrics()LRC timeline、翻译合并、无词返回 null

GitHub Pages Demo

每个 MusicConnect-* 仓库都应该提供自己的可运行 Demo，而不是只写 README。 Demo 页面用于验证动态加载、搜索、播放和配置错误提示。

html — demo import 
<script type="module">
  const mod = await import('../dist/index.js?_=' + Date.now());
  const connector = new mod.default();
  await connector.init(demoConfig);
</script>
!
需要凭证的连接器 QQMusic、Spotify 这类连接器 Demo 必须明确展示“未配置”状态和配置入口，不能静默失败。

版本与发版

正式宿主记录必须使用不可变 tag，不使用 @main、latest 或浮动 CDN。

cdn 
https://cdn.jsdelivr.net/gh/DancingMusic/MusicConnect-YourName@vX.Y.Z/dist/index.js
  • push main 前运行 npm test、npm run build 或仓库等价命令。
  • 版本 bump 后由 .github/workflows/release.yml 创建 vX.Y.Z tag 和 GitHub Release。
  • 宿主需要升级官方默认连接器时，只改 src/app/lib/music-connectors.ts 的 pinned URL。

宿主接入

DancingMusic 宿主会把“官方开发文档”和“官方规范”入口指向本页； 当前连接器的“官方 Demo”继续指向对应 MusicConnect-* 的 Pages Demo。

✓
推荐调试顺序 先在连接器仓 Demo 验证搜索和播放，再用宿主添加固定 tag URL 做动态加载 smoke test。

提交检查

  • README 写清用途、ID 前缀、API 端点、配置和固定 CDN URL。
  • 契约测试覆盖 metadata、search、stream success/null、playlist、lyrics。
  • dist/index.js 已构建并提交，Demo 页面用 cache-busted import。
  • 没有把任何实现加入主仓 submodule，也没有 import 宿主代码。
  • 需要宿主默认安装时，单独提交主仓 pinned tag 更新。
