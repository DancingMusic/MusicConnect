Overview

MusicConnect is a music data source connector library for DancingMusic. It provides a unified interface for connecting to multiple music platforms, allowing you to search tracks, retrieve metadata, stream audio, and fetch lyrics from any supported source through a single, consistent API.

Architecture

MusicConnect sits between MusicStoreSdk and the individual platform connectors, managed through a central registry:

┌─────────────────┐ │ MusicStoreSdk │ Application layer └────────┬────────┘ │ ▼ ┌─────────────────────────────┐ │ MusicConnectorRegistry │ Manages connector lifecycle │ ┌────────────────────────┐ │ │ │ register(connector) │ │ │ │ get(id) │ │ │ │ list() │ │ │ └────────────────────────┘ │ └────────┬────────────────────┘ │ ┌────┴─────┬──────────────┐ ▼ ▼ ▼ ┌────────┐ ┌────────┐ ┌────────────┐ │ Netease│ │ Custom │ │ Custom │ │Connector│ │Conn. A │ │ Conn. B │ └────────┘ └────────┘ └────────────┘ │ │ │ ▼ ▼ ▼ NetEase Platform A Platform B Cloud API API API

Key features:

  • Unified connector interface for all music platforms
  • Central registry for managing multiple connectors
  • Cross-platform search, streaming, and lyrics retrieval
  • Extensible architecture for adding custom connectors
  • Built-in NetEase Cloud Music connector

Quick Start

Install

npm install @dancingmusic/music-connect

Usage

import {
  MusicConnectorRegistry,
  NeteaseConnector,
} from '@dancingmusic/music-connect';

// Create the registry and register a connector
const registry = new MusicConnectorRegistry();
const netease = new NeteaseConnector({
  apiBase: 'https://netease-api.example.com',
});
registry.register(netease);

// Search for tracks
const connector = registry.get('netease');
const results = await connector.search('Jay Chou', { limit: 10 });

// Get stream URL for the first result
const stream = await connector.getStreamUrl(results[0].trackId);
console.log(stream.url, stream.format, stream.bitrate);

// Fetch lyrics
const lyrics = await connector.getLyrics(results[0].trackId);
lyrics.lines.forEach(line =>
  console.log(`[${line.time}]`, line.text)
);

MusicConnector Interface

All connectors must implement the MusicConnector interface. This is the contract that the registry and consuming code rely on.

interface MusicConnector {
  /** Connector metadata */
  readonly meta: MusicConnectorMeta;

  /** Search for tracks by keyword */
  search(
    keyword: string,
    options?: { limit?: number; offset?: number }
  ): Promise<MusicSearchResult[]>;

  /** Get track details by trackId */
  getTrack(trackId: string): Promise<MusicSearchResult | null>;

  /** Get a streamable URL for the track */
  getStreamUrl(trackId: string): Promise<MusicStreamInfo>;

  /** Get lyrics for the track */
  getLyrics(trackId: string): Promise<MusicLyrics | null>;
}
MethodSignatureDescription
metaMusicConnectorMetaRead-only connector metadata describing the platform
search(keyword, options?) => Promise<MusicSearchResult[]>Search for tracks by keyword with optional pagination
getTrack(trackId) => Promise<MusicSearchResult | null>Get detailed track information by ID
getStreamUrl(trackId) => Promise<MusicStreamInfo>Get a streamable URL and format info for the track
getLyrics(trackId) => Promise<MusicLyrics | null>Get time-synced lyrics; returns null if unavailable

MusicConnectorMeta

Metadata that describes a connector and its capabilities. Each connector exposes this via the meta property.

}

NeteaseConnector

The built-in connector for NetEase Cloud Music. It provides access to search, streaming, and lyrics from the NetEase platform.

Capabilities

  • search — Search tracks by keyword
  • stream — Get streamable audio URLs
  • lyrics — Fetch time-synced lyrics with translations

Configuration

OptionTypeDescription
apiBasestringBase URL for the NetEase API proxy
cookiestringNetEase login cookie for authenticated access (higher quality streams) optional
timeoutMsnumberRequest timeout in milliseconds (default: 10000) optional
quality'standard' | 'higher' | 'exhigh' | 'lossless'Preferred audio quality level (default: "higher") optional

Example

import { NeteaseConnector } from '@dancingmusic/music-connect';

const netease = new NeteaseConnector({
  apiBase: 'https://netease-api.example.com',
  cookie: 'MUSIC_U=your_cookie_here',
  quality: 'lossless',
});

console.log(netease.meta);
// {
//   id: 'netease',
//   name: 'NetEase Cloud Music',
//   icon: 'https://music.163.com/favicon.ico',
//   description: 'NetEase Cloud Music connector',
//   version: '0.1.0',
//   capabilities: ['search', 'stream', 'lyrics']
// }

const results = await netease.search('Mayday');
const stream = await netease.getStreamUrl(results[0].trackId);
const lyrics = await netease.getLyrics(results[0].trackId);

Creating a Connector

You can extend MusicConnect by implementing the MusicConnector interface. Follow these steps to create a custom connector for any music platform.

Step 1: Define the connector class

import type {
  MusicConnector,
  MusicConnectorMeta,
  MusicSearchResult,
  MusicStreamInfo,
  MusicLyrics,
} from '@dancingmusic/music-connect';

export class SpotifyConnector implements MusicConnector {
  readonly meta: MusicConnectorMeta = {
    id: 'spotify',
    name: 'Spotify',
    icon: 'https://spotify.com/favicon.ico',
    description: 'Spotify music connector',
    version: '0.1.0',
    capabilities: ['search', 'stream'],
  };

  constructor(private config: { clientId: string; clientSecret: string }) {}

  async search(keyword: string, options?: { limit?: number; offset?: number }) {
    // Call Spotify API and map results to MusicSearchResult[]
    return [];
  }

  async getTrack(trackId: string) {
    // Fetch a single track by its platform:id
    return null;
  }

  async getStreamUrl(trackId: string) {
    // Return stream info
    return { url: '', format: 'ogg', bitrate: 320 };
  }

  async getLyrics(trackId: string) {
    // Not supported — return null
    return null;
  }
}

Step 2: Register the connector

import { MusicConnectorRegistry } from '@dancingmusic/music-connect';
import { SpotifyConnector } from './spotify-connector';

const registry = new MusicConnectorRegistry();

registry.register(new SpotifyConnector({
  clientId: 'your-client-id',
  clientSecret: 'your-client-secret',
}));

// Now you can use it
const spotify = registry.get('spotify');
const results = await spotify.search('Daft Punk');

Step 3: Handle capabilities

If your connector does not support a capability (e.g. lyrics), declare only the supported ones in meta.capabilities and return null from the unsupported method. Consumers should check capabilities before calling:

const connector = registry.get('spotify');

if (connector.meta.capabilities.includes('lyrics')) {
  const lyrics = await connector.getLyrics(trackId);
} else {
  console.log('Lyrics not supported by this connector');
}

Connector ID Convention

To avoid ID collisions across different music platforms, all trackId values must follow the platform:id format.

Format

// Pattern: <platform>:<platform_specific_id>

"netease:1234567"     // NetEase Cloud Music track
"spotify:4uLU6hMCjMI75M1A2tKUQC"  // Spotify track
"qq:003aAYrk3GE"     // QQ Music track

Rules

  • The platform segment must match the connector's meta.id
  • The id segment is the platform's native track identifier
  • Connectors must generate trackId in this format when returning search results
  • Connectors must parse and strip the prefix when making platform API calls

Parsing example

// Utility to parse a trackId
function parseTrackId(trackId: string): { platform: string; id: string } {
  const idx = trackId.indexOf(':');
  if (idx === -1) throw new Error(`Invalid trackId format: ${trackId}`);
  return {
    platform: trackId.slice(0, idx),
    id: trackId.slice(idx + 1),
  };
}

const { platform, id } = parseTrackId('netease:1234567');
// platform === 'netease', id === '1234567'
